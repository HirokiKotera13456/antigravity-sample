# ビジュアルブランド監査プロトコル

このスキルは、プロジェクトの視覚的な一貫性を保つための「校正者」として機能します。

## 1. 監査項目
- **カラーパレット:** インラインカラーや未定義のTailwindカラーが使われていないか。
- **タイポグラフィ:** セリフ体とサンセリフ体の使い分けが意図通りか。
- **余白:** `p-12`, `py-32` などの大胆な余白が確保されているか。

## 2. 実行アクション
1. `src/` 配下のファイルを `grep_search` や `view_file` でスキャンする。
2. ガイドラインから逸脱している箇所をリストアップする。
3. 修正案（リファクタリング案）を作成し、ユーザーに提案する。

## 3. 監査のタイミング
- 大規模なコンポーネント追加の後。
- `npm run build` 前の最終確認。
- ユーザーから「デザインを整えて」と指示があった時。
